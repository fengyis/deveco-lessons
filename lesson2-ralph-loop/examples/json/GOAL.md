# 目标

仓库里有一个**半成品** JSON 解析器 `src/json.ts` 和它的测试 `test/json.test.ts`。
现在 `bun test` 是**红的**——它只认 `null/true/false`、非负整数、以及由它们组成的数组。
你的活是把它**补全成一个严格符合 RFC 8259 的 JSON 解析器**。

```ts
parse(text: string): unknown
```

合法输入返回对应的 JS 值;**任何不合法的输入都必须抛异常**(不能返回 undefined、不能容错)。

## 规格(RFC 8259,严格模式)

- **字面量**:`null` / `true` / `false`(仅小写)。
- **数字**:`-?(0|[1-9]\d*)(\.\d+)?([eE][+-]?\d+)?`,一个字符都不能多。
  `-0` 要保号;前导零、裸小数点、悬空指数、`NaN`/`Infinity`、十六进制一律拒绝。
- **字符串**:仅双引号;转义仅 `\" \\ \/ \b \f \n \r \t \uXXXX`;
  `\uXXXX` 支持代理对;**裸控制字符(U+0000–U+001F)非法**;非法转义、未闭合 → 抛。
- **数组 / 对象**:任意嵌套;键必须是字符串;尾随逗号、缺冒号、缺值、未闭合一律拒绝。
- **空白**:token 之间仅允许 空格/`\t`/`\n`/`\r`;**字符串内部原样保留**。
- **整篇**:顶层可为任意 value;空输入、纯空白、尾部多余内容拒绝。

规格没列到的边界,以 RFC 8259 严格语义为准 —— **QA 会逐条查边界,不会只查上面这几行**。

# 验收标准

- [ ] `src/json.ts` 导出 `parse(text: string): unknown`

# 红线

- **不许回归**:字面量 / 非负整数 / 数组这三类**开局就是绿的**,补全过程中不能弄挂。
- **不许删改测试**来把 `bun test` 弄绿。
- **不许调内建**:`JSON.parse` / `eval` / `new Function` 一律不算——这是从零实现的练习。
- **不许对着用例硬编码**(比如 `if (text === '...') throw`)。
